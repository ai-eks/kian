# 微信独立聊天渠道接入说明

## 配置结构

`settings.json` 中新增：

```json
{
  "chatChannels": {
    "weixin": {
      "enabled": false,
      "stateDir": "",
      "accountId": ""
    }
  }
}
```

字段说明：

- `enabled`: 是否启用微信独立聊天渠道
- `stateDir`: 微信适配器状态目录，留空时使用适配器默认目录
- `accountId`: 当前启用的微信账号 ID

## 主进程初始化

- `electron/main/services/chatChannel/weixinChannelService.ts`
  - 封装 `@kian/weixin-adapter`
  - 管理扫码登录、账号列表、轮询状态、错误状态、文本发送
- `electron/main/services/chatChannelService.ts`
  - 通过 `weixinChannelService.refresh()` 接入运行时
  - 监听微信入站消息并转给主 Agent
  - 将主 Agent 回复文本回发到微信

## Renderer 入口

设置页新增微信渠道 Tab，提供：

- 启用开关
- `stateDir` 配置
- 已保存账号选择
- 开始扫码登录
- 等待登录确认
- 当前轮询状态、实际状态目录、最近错误展示

## 当前已支持

- 微信作为独立 chat channel 接入，不复用 broadcast channel
- 基于 `@kian/weixin-adapter` 的二维码登录
- 本地账号保存与账号选择
- 长轮询收取微信消息
- 文本消息送入主 Agent
- 主 Agent 文本回复回发微信

## 当前未支持

- 图片、语音、视频、文件消息的完整收发链路
- 多账号同时并行轮询
- 更细粒度的登录生命周期展示
- 更强的异常重试与恢复策略

## 手动测试

1. 打开设置页 `Channels > 微信`
2. 填写或确认 `stateDir`
3. 点击“开始微信扫码登录”，用微信扫描二维码
4. 点击“等待微信登录确认”，确认账号写入成功
5. 选择 `accountId`，并启用微信渠道
6. 向该微信账号发送一条文本消息，确认消息进入主 Agent
7. 等待 Agent 回复，确认文本消息成功回发微信

## 备注

- 当前实现没有引入 `openclaw/plugin-sdk`
- 微信协议与宿主逻辑已分层：协议封装在 `@kian/weixin-adapter`，宿主接线在 Electron main services
